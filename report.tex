Отчёт является основным отчётным документом о результатах выполнения курсовой работы, должен содержать описания всех этапов работ с фиксацией содержательного наполнения. Содержание отчёта должно обеспечить воспроизводимость любого этапа работ и быть понятным стороннему читателю.

Отчёт формируется по правилам, определённым ГОСТ 7.32-2001 <<Отчёт о научно-исследовательской работе. Структура и правила оформления>>. Предпочтительный формат бумаги --- А4, ER--диаграммы, схемы базы данных допускается выполнять на форматах А3. Предпочтительное размещение листа --- вертикальное, для таблиц допускается горизонтальное. Все таблицы и рисунки должны быть подписаны и пронумерованы. Титульный лист считается, но не нумеруется.

Отчёт содержит:
\begin{enumerate}
\item{}Титульный лист.
\item{}Задание на курсовую работу.
\item{}Часть 1. Описание предметной области.
\begin{itemize}
\item{}текстовое описание предметной области;
\item{}определение основных понятий;
\item{}описание ролей;
\item{}графическое представление основного процесса;
\item{}иерархия объектов предметной области;
\item{}ER--диаграмма. Описание ER--диаграммы.
\end{itemize}
\item{}Часть 2. Проектирование схемы базы данных.
\begin{itemize}
\item{}описание атрибутов и отношений данных; 
\item{}графическое представление базы данных на русском языке;
\item{}графическое представление базы данных на английском языке с указанием числа записей в таблицах;
\item{}текстовое описание схемы базы данных.
\end{itemize}
\item{}Часть 3. Программирование схемы, подготовка исходных данных.
\begin{itemize}
\item{}исходный код программы на языке DDL для создания базы данных в СУБД MySQL. Текст приводится в приложении А;
\item{}исходный код программы на языке программирования высокого уровня для заполнения базы данных тестовыми данными. Текст приводится в приложении Б;
\item{}исходный код программы на языке DML с командами заполнения словарей. Текст приводится в приложении В. 
\end{itemize}
\item{}Часть 4. Программирование запросов к БД.
\begin{itemize}
\item{}описание запроса;
\item{}анализ эффективности выполнения запроса;
\item{}анализ результатов запроса. 
\end{itemize}
\item{}Выводы.
\end{enumerate}

Каждый раздел отчёта необходимо начинать с начала новой страницы. При необходимости графические материалы размещать на отдельных ``плавающих'' страницах. Размер и внешний вид графических данных должен обеспечить их читаемость, для этого необходимо выбирать гаммы цветов и текстуры заполнения, позволяющие надёжно различить части изображения.

Выполнение отчёта возможно в технологии  Microsoft Word for Windows или \LaTeX. 

Детальное описание содержания пунктов отчёта приведено в разделе~\Nameref{sec:report_entry}.
